Development and Automation of an O.R Database System

Introduction

This report presents the design, implementation, and automation of an object-relational database system using PL/SQL. The project involves analysing data requirements, designing an efficient schema, implementing key database functionalities, and automating operations.

Database Structure

The system is structured around key entities such as Practice, Programme, Guide, Guest, Team, Location, and Practice_Guide. Each entity is carefully modelled to ensure a well-structured relational database.

Entity-Relationship Diagram (ERD)

  • Guides manage multiple teams (one-to-many relationship).
  • Practice_Guide handles many-to-many relationships between guides and practices.
  • Guests are linked to programmes through a foreign key.
  • Locationes are stored in a structured format for guests, guides, and accommodations.

Key Functionalities

1. Queries

  • Retrieve and sort programmes by cost.
  • Count guests per programme.
  • Filter assigned practices by date.

2. Functions

A function DetermineEmploymentStatus evaluates a guide’s job status based on the number of teams they manage:

  • More than 5 teams → Full-time
  • 5 or fewer teams → Part-time

Another function generates a welcome message for guests dynamically.

3. Triggers

  • Enforce date-of-birth validation to prevent future dates.
  • Automatically update guide employment status.

4. Stored Procedures

  • Insert new guests and teams into the system.
  • Update guide employment status dynamically.

5. Cursors

Cursors iterate over guest records for dynamic data retrieval and analysis.

Testing and Validation

Various test cases ensured the system operates efficiently:

  • Table creation and structure validation.
  • Data insertion and update constraints.
  • Function accuracy for employment status.
  • Trigger execution for data integrity.
  • Performance validation of complex queries.

Automation Strategy

The automation strategy focuses on optimizing database operations through:

  • Stored procedures for standardization.
  • Triggers for enforcing business rules.
  • User-defined functions for real-time updates.
  • Efficient queries for data-driven decision-making.

Challenges and Solutions

  • Data Integrity: Enforced through constraints and triggers.
  • Complex Relationships: Managed via intermediary tables.
  • Query Performance: Optimized with indexing.

Conclusion

This project successfully demonstrates the development and automation of an object-relational database system, ensuring efficiency, scalability, and reliability. The combination of PL/SQL procedures, functions, triggers, and queries provides a robust framework for handling organisational data, laying a strong foundation for future expansion.
